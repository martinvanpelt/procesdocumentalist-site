Dit BPMN-template biedt een gestructureerde aanpak voor het modelleren van processen met behulp van Business Process Model and Notation (BPMN 2.0). Het doel is om:  

- Standaardisatie te waarborgen in de modellering van processen binnen {{organisatienaam}}.  
- Duidelijkheid te creëren over processtappen, beslissingen, rollen, en systemen.  
- Communicatie tussen stakeholders (management, uitvoerende teams, IT) te verbeteren.  
- Basis te leggen voor automatisering, analyse, en optimalisatie van processen.

#### 2. BPMN Basisstructuur

BPMN-diagrammen bestaan uit vier hoofdcategorieën van elementen:

1. Flow Objects (Stroomobjecten): Events, Activities, Gateways.
2. Connecting Objects (Verbindingen): Sequence Flows, Message Flows, Associations.
3. Swimlanes (Zwembanen): Pools en Lanes voor organisatie en verantwoordelijkheid.
4. Artifacts (Hulpmiddelen): Data Objects, Groups, Annotations.

#### 4. Trigger (Start Event)

Beschrijf hier wat het proces start. In BPMN wordt dit weergegeven als een Start Event (cirkel met dunne rand).


| Trigger                    | Type | Beschrijving                                                          | BPMN-symbool  | Verantwoordelijke |
| ------------------------------ | -------- | ------------------------------------------------------------------------- | ----------------- | --------------------- |
| [Bijv. "Ontvangst klantorder"] | Message  | Een klant plaatst een order via de webshop.                               | Start Event       | Sales Team            |
| [Bijv. "Tijdsgebaseerd"]       | Timer    | Het proces start op een vast tijdstip (bijv. dagelijkse batchverwerking). | Start Event Timer | IT-afdeling           |


Toelichting types Start Events:

- Message: Het proces start door een bericht (bijv. e-mail, API-call).
- Timer: Het proces start op een bepaald tijdstip of interval.
- Signal: Het proces start door een signaal (bijv. systeemalert).
- Conditional: Het proces start als een voorwaarde waar is (bijv. lage voorraad).

#### 5. Hoofdactiviteiten (Activities)

Beschrijf hier de hoofdactiviteiten van het proces. In BPMN worden activiteiten weergegeven als rechthoeken met afgeronde hoeken.


| Stap | Activiteit                        | Type | Beschrijving                                    | BPMN-symbool | Verantwoordelijke | Systeem/Tool | Duur   |
| -------- | ------------------------------------- | -------- | --------------------------------------------------- | ---------------- | --------------------- | ---------------- | ---------- |
| 1        | [Bijv. "Ontvangst klantorder"]        | Task     | Registratie van de klantorder in het ERP-systeem.   | Task             | Order Team            | ERP-systeem      | 5 minuten  |
| 2        | [Bijv. "Validatie klantgegevens"]     | Task     | Controle of klantgegevens compleet en correct zijn. | Task             | Order Team            | CRM-systeem      | 10 minuten |
| 3        | [Bijv. "Genereren productieopdracht"] | Task     | Omzetten van klantorder naar productieopdracht.     | Task             | Order Team            | ERP-systeem      | 15 minuten |


Toelichting types Activities:

- Task: Een enkele activiteit (bijv. "Validatie klantgegevens").
- Subprocess: Een groep van activiteiten die samen een deelproces vormen.
- Loop Task: Een herhalende activiteit (bijv. "Maandelijkse rapportage").
- Parallel Multi-Task: Meerdere taken die gelijktijdig worden uitgevoerd.

#### 6. Beslispunten (Gateways)

Beschrijf hier de beslissingspunten in het proces. In BPMN worden gateways weergegeven als diamanten.


| Beslissing                      | Type        | Beschrijving                                  | Opties | Gevolg per optie                                            | BPMN-symbool |
| ----------------------------------- | --------------- | ------------------------------------------------- | ---------- | --------------------------------------------------------------- | ---------------- |
| [Bijv. "Is de order compleet?"]     | Exclusive (XOR) | Controle of alle vereiste velden zijn ingevuld.   | Ja / Nee   | Ja: Doorgaan naar stap 4, Nee: Terug naar klant voor aanvulling | XOR Gateway      |
| [Bijv. "Is de voorraad voldoende?"] | Exclusive (XOR) | Controle of er genoeg producten op voorraad zijn. | Ja / Nee   | Ja: Doorgaan naar productie, Nee: Inkoop inschakelen            | XOR Gateway      |


Toelichting types Gateways:

- Exclusive (XOR): Slechts één pad kan worden gevolgd (bijv. "Ja" of "Nee").
- Inclusive (OR): Meerdere paden kunnen worden gevolgd (bijv. "A en B", "A en C").
- Parallel (AND): Alle paden moeten worden gevolgd (bijv. "A en B en C").
- Event-based: De keuze is afhankelijk van een event (bijv. ontvangst van een bericht).

#### 7. Uitzonderingen (Intermediate Events)

Beschrijf hier wat er gebeurt als het proces niet volgens de hoofdflow verloopt. In BPMN worden uitzonderingen weergegeven als Intermediate Events (cirkels met dubbele rand).


| Uitzondering                    | Type | Beschrijving                      | Oorzaak                                       | Afhandeling                                             | BPMN-symbool |
| ----------------------------------- | -------- | ------------------------------------- | ------------------------------------------------- | ----------------------------------------------------------- | ---------------- |
| [Bijv. "Onvolledige klantgegevens"] | Error    | Foutmelding bij ontbrekende gegevens. | Klant heeft niet alle verplichte velden ingevuld. | Terugsturen naar klant voor aanvulling.                     | Error Event      |
| [Bijv. "Systeemstoring"]            | Signal   | Onderbreking door systeemfout.        | ERP-systeem is niet beschikbaar.                  | Handmatige registratie in Excel, later overzetten naar ERP. | Signal Event     |


Toelichting types Intermediate Events:

- Message: Een bericht ontvangen tijdens het proces.
- Timer: Een tijdsgebaseerde onderbreking (bijv. time-out).
- Error: Een fout treedt op in het proces.
- Signal: Een signaal ontvangen (bijv. systeemalert).

#### 8. Eind Event (End Event)

Beschrijf hier hoe het proces eindigt. In BPMN wordt dit weergegeven als een cirkel met dikke rand.


| Eind Event               | Type | Beschrijving                                | BPMN-symbool  |
| ---------------------------- | -------- | ----------------------------------------------- | ----------------- |
| [Bijv. "Order bevestigd"]    | Message  | De orderbevestiging is verstuurd naar de klant. | End Event Message |
| [Bijv. "Proces geannuleerd"] | Error    | Het proces is geannuleerd door een fout.        | End Event Error   |


Toelichting types End Events:

- Message: Het proces eindigt met het versturen van een bericht.
- Error: Het proces eindigt met een foutmelding.
- Terminate: Het proces eindigt onmiddellijk (geen verdere actie).

#### 11. BPMN Diagram (Visuele Weergave)

Voeg hier een visuele weergave van het proces toe in BPMN-notatie. Gebruik Mermaid voor een eenvoudige weergave in markdown, of verwijz naar een BPMN-tool (bijv. Camunda, Signavio, Lucidchart).

Voorbeeld in Mermaid (BPMN-achtig):

```mermaid
%% BPMN voorbeeld - vervang door eigen diagram
bpmnDiagram
    %% Swimlanes
    pool "Orderverwerking" {
        lane "Order Team" as order_team {
            startEvent "Ontvangst klantorder" as start
            userTask "Validatie klantgegevens" as validatie
        }
        lane "Productie Team" as productie_team {
            userTask "Genereren productieopdracht" as productie
        }
    }

    %% Flow
    start --> validatie
    validatie --> gateway1

    %% Gateway
    exclusiveGateway "Is de order compleet?" as gateway1
    gateway1 -->|Ja| productie
    gateway1 -->|Nee| userTask "Aanvullen klantgegevens" as aanvullen
    aanvullen --> validatie

    %% End Event
    productie --> endEvent "Order bevestigd" as end
```

Tip:

- Gebruik een dedicated BPMN-tool (bijv. Camunda, Signavio) voor complexe diagrammen met alle BPMN-symbolen.
- Exporteer het diagram als afbeelding en voeg deze toe aan het document.

#### 13. Tips voor Effectieve BPMN-Modellering

- Gebruik de juiste symbolen: Zorg dat elk element (Start Event, Task, Gateway) het juiste BPMN-symbool heeft.  
- Houd het eenvoudig: Begin met een hoofdstroom en voeg luego beslissingen en uitzonderingen toe.  
- Gebruik Swimlanes: Maak duidelijk wie verantwoordelijk is voor elke stap.  
- Documenteer alles: Zorg dat alle stappen, beslissingen, en uitzonderingen zijn gemodelleerd.  
- Valideer met stakeholders: Laat het diagram reviewen door proceseigenaren en uitvoerende teams.  
- Gebruik tools: Maak gebruik van BPMN-tools (bijv. Camunda, Signavio) voor complexe diagrammen.  
- Houd het actueel: Update het diagram bij wijzigingen in het proces.
